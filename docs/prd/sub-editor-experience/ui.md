# UI 规范：编辑器体验

## 0. 文档信息

- Sub ID：SUB-002；状态：草稿；类型：UI 型。

## 1. 信息架构与入口

参考应用使用固定侧边导航进入 `/inline`、`/chat`、`/both`。每页保留编辑器主区域；模型选择、运行状态和错误反馈置于页级控制区。聊天面板仅在 chat/both 路由显示。

## 2. 视觉与组件约定

- 复用 `@workspace/ui` 的按钮、菜单、表单和提示组件，BlockNote 风格限定在 editor 包作用域内。
- 编辑器、工具栏、slash 菜单和 AI 建议状态必须有一致的间距、焦点环与禁用样式。
- busy 状态使用文字、图标和 `aria-live`，不得只依赖颜色。

## 3. 通用交互

- AI 正在运行时，另一类 AI 入口禁用并说明原因；完成、失败、中止或拒绝后立即恢复。
- 模型加载、空列表、认证失败和流错误提供可操作提示，错误不得泄露服务端细节。
- 编辑器焦点在菜单关闭、接受/拒绝或聊天工具执行后回到合理编辑位置。

## 4. 国际化、可访问性与响应式

默认 zh-CN，字符串由可替换字典提供；菜单、面板、按钮均可键盘导航。窄屏时侧边栏与聊天面板可折叠，编辑器始终优先保留可编辑面积。

## 5. 验收标准

- 三个路由入口可被键盘、屏幕阅读器和指针操作。
- 空、加载、错误、AI busy 和无权限状态有明确文本。
- 现代桌面浏览器与窄屏不会遮挡编辑内容或关键操作。

## 1. 信息架构与入口

对话入口为 `TapNoteChatPanel` 组件,由集成方按需放置在任意区域(右侧、左侧、浮动、独立路由等均可)。`@tap-note/ai-chat` 包只提供位置无关的 ChatPanel 本体与 `createTapNoteChatAssistant` 入口,不提供抽屉开关/布局容器组件。本 feat 包约束:ChatPanel 最小宽度 320px。

聊天面板由消息列表、上下文三态选择(选区/全文/无)、输入框、发送/中止按钮、工具调用结果气泡构成。

```text
创作者打开 ChatPanel(由集成方放置)
  -> 选择上下文模式(选区/全文/无)
  -> 输入消息 -> 发送
  -> 流式渲染 text 与 tool part 状态
  -> 工具执行后气泡展示操作类型与结果
  -> 多轮继续
```

```mermaid
flowchart LR
  P[ChatPanel 集成方放置] --> CTX[上下文三态]
  CTX --> IN[输入框 发送/中止]
  IN --> ML[消息列表]
  ML --> TB[工具结果气泡]
  TB --> MUL[多轮继续]
```

│   ◔ updateBlock  输入中   #b2  │  ← 消息列表(可滚动)
│                                  │
│  ┌ ✓ updateBlock 已更新 #b2 ─┐ │  ← 工具结果独立气泡

- header 固定置顶,含面板标题与关闭按钮。

- 消息列表占据中部可滚动区域,用户消息右对齐、AI 消息左对齐。
- 输入区固定置底,显示当前模式(发送 / 中止 / busy 置灰)。

### 1.3 UI 状态流

```text
                          ┌─────────────────┐
                          │      idle       │  ← 默认,输入框可用
                          └────────┬────────┘
                                   │ busy.acquire("chat") 成功
                                   ▼
                          ┌─────────────────┐
              ┌─────────── │    sending      │  ← 请求中
              │            └────────┬────────┘
              │                     │ 收到首个 chunk
              │                     ▼
              │            ┌─────────────────┐
              │            │   streaming     │  ← 文本流式 + tool-call 输入中
              │            └────────┬────────┘
              │                     │ tool-call 完整,客户端 execute
              │                     ▼
              │            ┌─────────────────┐
              │  仅重试    │  tool-pending   │  ← 单 tool-call execute
              │ ┌──────────│  (单操作)       │
              │ │          └────────┬────────┘
              │ │                   │ execute 完成
              │ │                   ▼
              │ │          ┌─────────────────┐
              │ │          │   completed     │  ← 一轮结束,可继续多轮
              │ │          └────────┬────────┘
              │ │                   │ 用户继续输入
              │ │                   ▼
              │ │          ┌─────────────────┐
              │ └──────────│  sending(下一轮)│
              └───────────►│                 │
                          └─────────────────┘

  任一阶段冲突 / 前置失败 / 流中断:
              ┌─────────────────┐
              │     error      │  ← 显示可重试气泡
              │  revision 冲突 │     [仅重试该操作]
              │  / 前置失败    │     / [中止当前轮]
              │  / 流中断      │
              └─────────────────┘
  busy.release 在 completed / error / 中止 时触发
```

- 与 inline 共享 busy:进入 `sending` 前 `busy.acquire("chat")`,失败则输入框置灰(见 §4.2)。
- 「仅重试该操作」用最新 `documentRevision` 重新 execute 当前 `toolCallId`,不重发整轮、不破坏已成功操作(tech.md §6)。
- 「中止当前轮」停止流式,保留已成功 tool-call 结果,触发 `busy.release`。

## 2. 通用交互与视觉

- 上下文三态明确显示「引用选区 / 引用全文 / 未引用」（SUB-003 ui.md §2）。
- 截断提示：引用全文超预算时显示「文档已截断：共 N 块，此处含前 M 块」；>2× 改发大纲时说明。
- 工具状态：显示「工具输入中 / 成功 / 失败 / 冲突」状态（SUB-003 ui.md §2）。
- 操作类型以文字 + 图标表达（如「已插入块」「已更新块」「已删除块」），不能仅以颜色区分。
- busy 时另一入口（inline slash/按钮）禁用并文字说明；chat 进行中时自身输入框可中止。
- 不因一个编辑器会话阻止其他会话（SUB-003 ui.md §2）。
- 复用 `@workspace/ui` 控件。

## 3. 字段、操作、校验与反馈

- 上下文选择：单选三态（选区/全文/无）。
- 输入框：消息文本；`Enter` 发送（Shift+Enter 换行）；空则发送禁用。
- 发送：`busy.acquire("chat")` 成功后发送；失败则输入框置灰。
- 中止：流式中显示中止按钮，停止当前轮。
- 工具气泡：展示工具名、目标块、输入中/成功/失败/冲突状态；冲突提供「重试」。
- 校验：选区超 4K 拦截提示；工具输入 Zod 校验失败展示错误。

### 3.1 上下文三态 segmented control

默认 `无`,切换到 `选区`/`全文` 时编辑器侧保留高亮直到发送(见 §3.4),消息气泡携带引用 chip:

```text
默认(无):
┌──────────────────────────┐
│ 引用: [ 选区 | 全文 |•无] │   • = 当前选中
└──────────────────────────┘   不发 documentState,不暴露 getDocumentSnapshot

选区:
┌──────────────────────────┐
│ 引用: [•选区 | 全文 | 无] │   选区 2 块 / 约 0.8K tokens ✓
└──────────────────────────┘
   ⓘ 选区超 4K,请减少或改用「引用全文+指令」     ← 超限时拦截提示

全文(预算内):
┌──────────────────────────┐
│ 引用: [ 选区 |•全文 | 无] │   全文 12 块 / 约 3.2K tokens ✓
└──────────────────────────┘

全文(超预算,截断):
┌──────────────────────────┐
│ 引用: [ 选区 |•全文 | 无] │   全文 47 块 / 约 11K tokens
└──────────────────────────┘
   ⓘ 文档已截断:共 47 块,此处含前 18 块

全文(>2× 预算,改发大纲):
┌──────────────────────────┐
│ 引用: [ 选区 |•全文 | 无] │   全文 120 块 / 约 38K tokens
└──────────────────────────┘
   ⓘ 文档过长,已发送结构化大纲
```

### 3.2 消息气泡类型

用户消息(右对齐)携带上下文引用 chip:

```text
                       ▌用户:把第二点展开成段落
                         § 选区 2 块              ← 引用 chip
```

AI 文本流式(左对齐),光标 `◌` 表示流式进行中:

```text
▌AI
│ ◌ 正在思考...
└
```

工具调用状态(嵌入 AI 消息内,与 AI 文本同属一条 UIMessage.parts):

```text
▌AI
│ 已分析选区。
│ ◔ updateBlock  输入中...   #b2   ← 旋转/输入中
│ ◑ updateBlock  输入中...   #b3
│ ◐ updateBlock  输入中...   #b4
└
```

工具结果独立气泡(在 AI 消息下方独立展示成功 / 冲突 / 前置失败):

```text
成功:
┌ ✓ updateBlock  已更新块    #b2 ─┐
│  目标:第二点                  │
│  [跳转到该块]                  │
└────────────────────────────────┘

版本冲突(可重试):
┌ ⚠ updateBlock  版本冲突     #b3 ─┐
│  文档已变更,该块当前 revision=8 │
│  期望 revision=5                 │
│  [仅重试该操作]                  │  ← 用最新 revision 重试当前 toolCallId
└──────────────────────────────────┘

前置条件失败(可重试):
┌ ⚠ updateBlock  目标块不存在 #b4 ─┐
│  块 #b4 已被删除                 │
│  [仅重试该操作]                  │
└──────────────────────────────────┘
```

错误气泡(流中断 / 认证失败,不暴露服务端堆栈):

```text
▌AI ✗
│ 流式中断,可重试或继续多轮。
└

▌AI ✗
│ 认证已过期,请重新登录。
└
```

## 4. 加载、空状态、错误状态与权限状态

- 加载：流式首块出现前显示思考指示。
- 空状态：无消息时输入框为主，提示可用上下文模式。
- 错误：流失败/工具失败展示可行动提示；认证失败提示重新认证；不暴露服务端堆栈。
- 权限/busty：另一 AI busy 时输入框置灰并说明原因。
- 冲突：revision/前置条件冲突气泡展示「版本冲突，可重试」。

## 5. 国际化、可访问性与响应式

- 默认 zh-CN，操作结果/错误/截断提示可由字典替换（SUB-003 ui.md §3）。
- 聊天面板支持焦点陷阱、`Escape` 关闭、发送后焦点恢复、`aria-live` 流状态播报。
- 不引用状态不误导用户为已发送文档（SUB-003 ui.md §4）。
- 窄屏(<768px)的具体呈现方式(底部 sheet / 抽屉 / 全屏)不在本 feat 范围内,由集成方按宿主应用实现;本 feat 只保证桌面布局(§1.1)与 `/both` 共存(§5.1)在现代桌面浏览器下可用,且 inline 审阅控制不被遮挡。

### 5.1 `/both` 路由:inline + chat 共存(apps/web demo example)

## 6. UI 验收标准

- 用户能通过键盘完成打开、选上下文、输入、发送、中止、多轮（§16 item 8 可访问性）。
- 上下文超限、认证/流失败、revision 冲突、工具失败均有可行动提示（§16 item 6/7/10）。
- 每个对话工具调用显示目标与结果；不引用状态不误导为已发送文档（§16 item 6）。
- 选区超 4K 前端拦截提示；引用全文超预算显示截断标记（§16 item 9）。
- 现代桌面浏览器不遮挡编辑内容或接受/拒绝控制。

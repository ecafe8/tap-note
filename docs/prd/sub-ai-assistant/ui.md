# UI 规范：AI 助手

## 0. 文档信息

- Sub ID：SUB-003；状态：草稿；类型：混合型。

## 1. 信息架构

内联入口包括 slash 项、选区工具栏按钮和 AIMenu；对话入口为可开合的 `TapNoteChatPanel`。聊天面板由消息、上下文三态选择、输入框、发送/中止和操作结果气泡构成。

## 2. 通用交互与视觉

- inline 依次可见输入、思考、流式建议、审阅、错误；审阅状态必须提供接受、拒绝和快捷键。
- chat 明确显示“引用选区/全文/未引用”、截断提示、工具输入中/成功/失败/冲突状态。
- busy 时另一入口禁用并保留原因；不因一个编辑器会话阻止其他会话。
- 复用 `@workspace/ui` 控件；操作类型以文字+图标表达，不能仅以颜色区分。

## 3. 国际化、可访问性与响应式

默认 zh-CN，操作结果及错误可由字典替换。菜单和聊天支持焦点陷阱、Escape 关闭、发送后焦点恢复、`aria-live` 流状态。窄屏中聊天作为可折叠抽屉，不能覆盖接受/拒绝控制。

## 4. 验收标准

- 用户能通过键盘完成唤起、输入、发送、取消、接受和拒绝。
- 上下文超限、认证/流失败、revision 冲突和工具失败均有可行动提示。
- 每个对话工具调用显示目标与结果；不引用状态不误导用户为已发送文档。
